Informações para geração de proventos para ações

Funcionamento padrão

O processo envolve 2 usuários, um para a etapa de leitura, e outro para realizar a validação.
1
Reserva
Usuário 1 reserva pendência
2
Leitura
Usuário 1 realiza leitura
3
Reserva
Usuário 2 reserva pendência
4
Validação/Recusa
Usuário 2 realiza validação/recusa

Selecione um passo para detalhá-lo

Reservas
  • Para que um usuário possa realizar a leitura de um documento, é necessário reservar sua pendência.
    • Na tela de listagem de pendências, clique no botão RESERVAR para ser o responsável pela pendência.
    • Cada usuário pode reservar para si um máximo de 20 pendências
  • Caso a pendência já possua um responsável, seu nome irá aparecer na coluna mais à direita, onde ficaria o botão RESERVAR
  • Para não ser mais o responsável por uma pendência, basta clicar em CANCELAR na tela de listagem de pendências.
  • Não é possível ser o responsável por uma pendência de validação de um documento para o qual já tenha realizado a leitura.
Ações
  • As ações podem ser chamadas de papéis nos documentos.
  • As categorias possíveis de serem encontradas nos documentos são:
    • ON: ações ordinárias, representadas pelo ticker com um 3. Exemplo: BBAS3
    • PN: ações preferenciais, representadas pelo ticker com um 4. Exemplo: PETR4
    • PNA: ações preferenciais classe A, representadas pelo ticker com um 5. Exemplo: VALE5
    • PNB: ações preferenciais classe B, representadas pelo ticker com um 6. Exemplo: ELET6
    • PNC: ações preferenciais classe C, representadas pelo ticker com um 7. Exemplo: CELP7
    • PNC: ações preferenciais classe D, representadas pelo ticker com um 8. Exemplo: AMRI8
    • UNIT: grupos de ações ordinárias e preferenciais, representadas pelo ticker um 11. Exemplo: TAEE11
  • É importante lembrar que UNITS por enquanto não estão sendo adicionadas, caso seja encontrado um documento descrevendo proventos para UNIT, cancelar leitura.
  • Caso o documento mencione uma categoria de ação que não esteja disponível no sistema, avisar o administrador.
Proventos
  • Os tipos de proventos a serem encontrados nos documentos são:
    • Dividendos: proventos em dinheiro, descritos como uma quantia (em reais) por ação. O imposto de renda é retido na fonte.
    • Juros sobre Capital Próprio, (JCP ou JSCP): proventos em dinheiro, descritos como uma quantia (em reais) por ação. O imposto de renda (15%)é retido no pagamento do acionista.
    • Proventos em ações: proventos em ações, descritos como uma porcentagem ou proporção da quantidade de ações do acionista.
      • Também serão inseridos como proventos em ação, desdobramentos e agrupamentos de ações.
      • Desdobramentos são eventos onde a quantidade total de ações é multiplicada, num desdobramento de 1 para 5 por exemplo, o acionista tem sua posição aumentada para 5 vezes a posição anterior, indo de 100 ações para 500.
      • Agrupamentos são o oposto, a quantidade total de ações é dividida, num agrupamento de 5 para 1 o acionista vê sua posição ir de 100 para 20.
      • Caso seja encontrado um documento que anuncie um desdobramento ou agrupamento, avisar o administrador.
    • Podem aparecer também proventos de subscrição, esses eventos não serão considerados pelo sistema pois são emissões de novas ações a um preço próximo do já negociado, para as quais os atuais acionistas tem preferência na compra.
  • É importante lembrar que UNITS por enquanto não estão sendo adicionadas, caso seja encontrado um documento descrevendo proventos para UNIT, cancelar leitura.
  • Caso o documento mencione uma categoria de ação que não esteja disponível no sistema, avisar o administrador.
  • Caso o documento mencione uma alteração de provento, sem descrever todos os valores, avisar o administrador.
Valores
  • Os valores devem ser inseridos no sistema com todas as casas decimais mostradas no documento.
  • Deve sempre ser inserido o valor recebido por cota.
    • Para Juros sobre Capital Próprio, inserir o valor bruto, sem o desconto de IR.
  • Caso sejam descritos dois valores para uma ação, criar dois proventos. Ex: dividendos previstos e dividendos adicionais. Não somar os valores em um dividendo único.
  • Para os proventos em ações que possuam pagamento de frações, pegar o valor da fração descrito.
  • Caso seja encontrado um provento que mencione pagamento de rendimento, avisar o administrador.
  • Caso o pagamento seja parcelado, cada parcela deve ser adicionada com um provento separado.
Datas
  • As datas inseridas no sistema são:
    • Data EX: data a partir da qual não é mais considerada a posição acionária do investidor.
    • Data de pagamento: data na qual o investidor recebe os proventos indicados.
    • Data de pagamento de frações: data na qual o investidor recebe o valor fracionário correspondente a proventos em ações (apenas para o caso de proventos em ações).
  • Caso o documento não descreva a DATA EX, pegar o próximo dia útil após a DATA BASE (também chamada de DATA DE CORTE), geralmente mencionada como data na qual os detentores da ação fazem jus ao provento.
Razões para um documento ser desconsiderado sem leitura
  • Documento é uma ata de reunião/assembléia.
  • Documento possui mais de 10 páginas.
  • Documento é uma convocação para reunião/assembléia.
  • Documento é um Aviso a debenturistas.
Validação
  • Verificar se dados dos proventos criados na leitura batem com os do documento.
  • Caso o documento seja uma errata ou alguma alteração sobre um provento, selecionar o provento ao qual ele se relaciona. Será criada uma nova versão para o provento.
    • Verificar se entre os proventos próximos listados, não se encontra algum com mesma DATA EX e TIPO DE PROVENTO, pois pode ser uma alteração de provento já cadastrado.
  • Proventos que são alterações de proventos anteriores, que ainda não foram cadastrados, devem ter validação adiada.
  • Se os dados de algum dos proventos criados na etapa de leitura não estiver condizente com o texto do documento, recusar. Os passos são:
    • Selecionar o botão Recusar.
    • Preencher o campo de texto Motivo de recusa que irá aparecer.
    • Confirmar no botão Confirmar recusa.
  • Ao ser recusado, o documento volta como uma pendência de leitura para o usuário que foi responsável por sua leitura.
